# 知识导航、数学元数据与词条移动

Wikist v0.8 将词条之间的关系拆成两层：正文中的 `[[Wiki 链接]]` 形成可查询的链接索引；可选 Markdown front matter 则表达数学写作中的前置知识、规范名称、记号、分类和主题。两层都不依赖图数据库，也不会改变词条仍以 Markdown 文件保存的边界。

## 可移植的数学元数据

下列字段可在词条编辑器的“知识上下文”区域填写，也可随 Wikist JSON 导入导出：

```yaml
categories: [数学/代数/群论]
topic: 数学/代数/群论
canonicalNames: [Group theory, 群理论]
classifications: [MSC 20-XX, ACM CCS: Mathematics of computing]
prerequisites: [set-theory]
relatedPages: [ring-theory, representation-theory]
notation: ["G | 群 | 全文", "H | G 的子群 | 子群部分"]
```

- `categories`：面向读者浏览的分类路径；`数学/代数/群论` 会自动产生“数学”“数学/代数”和“数学/代数/群论”三级入口。
- `topic`：跨分类的学科脉络，可用同样的斜杠路径。
- `canonicalNames`：规范中文、英文或历史名称，不参与 slug 解析。
- `classifications`：MSC、ACM CCS 或站点自定义分类标识。
- `prerequisites`、`relatedPages`：正式词条 slug；阅读页会显示为可点击的知识上下文链接。
- `notation`：每项采用 `符号 | 含义 | 适用范围`。省略最后一段也可。

这些字段均为可选。普通词条不填写时不会出现空面板，也不要求写作者先完成表单才能保存正文。

## 分类与主题页

前台入口：

- `#/category`：根分类目录。
- `#/category/<分类路径>`：分类落地页，展示子分类、直接归类词条和质量分布。
- `#/topic` 与 `#/topic/<主题路径>`：同样的轻量主题树。

父级统计包含其所有后代词条，但分类详情中的“词条”只显示直接归入当前分类的内容，子分类负责继续展开。重定向词条不计入分类和主题统计，避免一次移动带来重复计数。

## 词条内知识链接分页

阅读页的“知识链接”面板保留在词条正文下方：

- 反向链接与正文链接独立分页，默认每页 8 条。
- 翻页仅请求对应列表的当前页，不跳转词条、不重渲染正文。
- 总数和页码来自 SQLite 的增量链接索引；保存、导入、恢复、删除时只同步受影响词条。

接口为 `GET /api/pages/<slug>/links`，可使用 `backlinksPage`、`outgoingPage` 与 `limit` 参数。`limit` 限制在 4 到 40 之间，前台默认使用 8。

## 安全移动词条

资深编辑和管理员可在词条“权限与治理”页使用“移动词条”。移动以新 slug 为目标，并可选择保留旧 slug 的 Markdown 重定向。

一次移动会：

1. 移动 `content/pages/` 下的 Markdown、修订目录和已审阅稳定快照目录。
2. 迁移 SQLite 中的权限、编辑记录、审核记录、评论、评分、收藏、译文、关注、别名、消息链接与链接索引。
3. 重写其他词条正文中的 `[[旧 slug]]` / `[[旧 slug|显示名]]`，以及前置知识、相关词条、重定向目标和消歧指向。
4. 保留旧 slug 时，生成一个明确指向新 slug 的轻量重定向词条。

目标 slug 已存在实体词条、修订/稳定快照目录、别名，或残留协作数据时，移动会被拒绝，不会静默合并两套历史。移动后请在版本审阅队列中按正常流程审阅新版本。

## 语言感知链接与译文审核

当读者以非源语言阅读词条时，正文中的词条链接会携带相同语言参数。目标词条已有已发布译文时直接显示该译文；尚未发布时明确回退到源文，而不是跳入编辑器。

译文状态为 `draft`、`review`、`changes_requested`、`published`。只有 `published` 译文对普通读者可见；译者和资深编辑可在翻译工作台查看未发布工作区。完整规则见 [译文审核](TRANSLATION_REVIEW.md)。

## 回归检查

```powershell
npm run check:v08
```

该检查使用临时 Markdown 目录和临时 SQLite 数据库，覆盖数学元数据持久化、三级分类/主题、词条移动、历史与稳定快照迁移、链接重写、关注与收藏迁移、译文审核，以及 10 条反向链接的独立分页。
