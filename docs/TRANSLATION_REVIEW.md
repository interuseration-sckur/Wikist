# 轻量译文审核

Wikist 的文章审核和译文审核是两条独立工作流。文章稳定版本使用 `content/reviewed/` 加 SQLite 指针；译文仍以 SQLite 中的小型记录保存，以便同一源词条拥有多个语言版本而不复制源 Markdown。

## 状态流

```text
draft -> review -> published
              -> changes_requested -> draft / review
```

- `draft`：译文进度不足 95%，仅译者和资深编辑在工作台中可见。
- `review`：译者保存到 95% 或以上后自动进入待审状态。
- `changes_requested`：资深编辑要求补充或修订，并留下可见的审核意见。
- `published`：资深编辑或管理员通过后，普通读者在语言切换中可阅读。

每次译者再次保存，状态会重新按照当前完成度计算，并清除上一次审核人、审核时间和审核意见，避免旧批准误覆盖新内容。

## 权限与界面

- 已加入翻译社区的登录用户可以创建和保存自己负责语言的译文。
- `senior_editor` 与 `admin` 可在翻译工作台通过或要求修改任何译文。
- 译者只能在自己的工作区查看未发布译文；审核者也可查看。普通读者只能读取 `published`。
- 审核意见使用现有的审计日志记录 `translation.review`，并显示审核人、审核时间与说明。

阅读页的语言切换只负责阅读：目标语言没有已发布译文时，页面显示源文和“译文尚未发布”的明确提示。编辑与审核始终在 `#/translate/<slug>?lang=<language>` 中完成。

## 存储与迁移

`page_translations` 保存语言、标题、摘要、源文快照、译文 Markdown、进度、状态、译者和审核信息。它不替代原始 Markdown 词条。

词条移动时，译文的 `page_slug` 与其中的 Wikist 链接会同步更新。备份与恢复继续使用现有 SQLite 用户数据选项，因此升级到 v0.8 不要求单独导出译文。

## v0.9 翻译质量层

v0.9 在不改变 Markdown + SQLite 默认部署方式的前提下，新增审核门控的翻译记忆、社区术语表和源文段落变更提示。

- 只有 `published` 译文会按空行段落配对进入 `translation_memory`；草稿、待审和要求修改的译文不会污染建议库。
- 源文快照与当前 Markdown 使用规范化段落哈希比较。翻译工作台明确提示新增和移除段落，但不会自动改写译文。
- `#/translation-glossary` 提供按语言方向检索与分页；翻译社区成员可浏览，`senior_editor` 与 `admin` 可维护推荐译法、记法、说明和避免使用的译法。
- 词条移动会同时迁移记忆记录的所属 slug，并修复其中的 Wikist 链接。

完整数据模型、接口和测试边界见 [翻译质量层](TRANSLATION_QUALITY.md)。Wikist 仍不实现通用软件本地化平台、外部机器翻译服务或语义向量检索；这些能力如有必要，应保持为可选、可审计的后续扩展。
