# Wikist 站点升级日志

## 2026-07-11 - v0.10.0 写作组织与社区审阅

- 新增写作组织：`writing_organizations`、成员、任务、讨论回复、社区投票和共识记录均自动创建在 Passport SQLite 中，不改变词条 Markdown、修订快照或备份格式。
- 组织创建者自动成为 `owner`；成员可承担 `member`、`writer`、`translator`、`reviewer`、`coordinator` 与 `owner` 角色。协调者可以管理成员、任务和讨论；最后一个活跃所有者不会被降级。
- 协作任务可绑定既有词条，分为撰写、翻译、审阅三类；成员可认领和提交待审，组织页对成员、任务和讨论均使用分页。
- 组织讨论支持 Markdown、回复、公告、社区决议、结论状态和组织内站内信提醒；它不是独立论坛服务，也不会新增外部依赖。
- 对已建立审阅任务的词条或译文，组织审阅者可对当前快照投票。达到组织阈值后，词条会建立稳定版本，译文会发布或转为需要修改；结论写入现有审阅记录并保留组织共识身份。
- `#/translate/<slug>` 改为目标语言选择页；已有进度、源文变更和自定义语言都可见，实际翻译路由为 `#/translate/<slug>?lang=<language>`，不再默认英文。
- 新增 `docs/WRITING_COMMONS.md` 和 `npm run check:v10`。框架升级为 `0.10.0`，核心前端资源升级为 `wikist-core-20260711-76`；使用 CDN 时请同步新的 `/assets/app.js`、`/assets/styles.css` 与安装器资源。

## 2026-07-11 - v0.9.0 翻译质量层与权限工作台

- 词条“权限与治理”页面改为响应式双栏工作台：访问策略、移动/重命名在桌面端并列，归档删除横跨完整宽度；窄屏自然回落为单列。所有新控件均使用 Wikist 自有深浅主题变量。
- 新增 SQLite `translation_memory` 与 `translation_glossary` 表。数据库初始化会自动创建，不需要手动迁移；包含用户数据的既有站点备份会自动携带它们。
- 只有资深编辑或管理员审核并发布的译文才会进入翻译记忆。按 Markdown 空行段落匹配并保存规范化源文哈希，草稿、待审、要求修改的译文不会进入建议库。
- 翻译工作台会比较保存时的源文快照与当前词条，显示有界的新增/移除段落提示；命中记忆和术语可插入当前 Markdown 光标位置，但不会自动保存或越过审核。
- 新增 `#/translation-glossary?source=zh-CN&target=en`：翻译社区成员可按语言方向分页检索术语；资深编辑和管理员可维护推荐译法、记法、说明、避免使用项和停用状态。
- 词条移动时会同步迁移翻译记忆所属 slug，并修复记忆文本中的 Wikist 链接。
- 新增 `npm run check:v09`；更新程序会执行这一检查。框架升级为 `0.9.0`，核心前端资源升级为 `wikist-core-20260711-74`。使用 CDN 时请同步 `/assets/app.js`、`/assets/styles.css`、安装器资源，并清理旧缓存。

- 新增语言感知词条链接：非源语言阅读时会沿用语言参数；目标译文未发布时明确显示源文回退。

## 2026-07-11 - SQLite FTS5 持久搜索索引

- 新增可选 SQLite FTS5 持久全文索引，复用 Passport 现有数据库；无需 Elasticsearch、独立服务或启动时遍历全量 Markdown。
- 词条创建、保存、恢复和删除会经由 PageStore 变更流只维护受影响的索引记录；既有链接索引和关注通知仍沿用原有的增量写入路径。
- 在管理员主动完成历史回填前、运行时不支持 FTS5 时，或需要短语/模糊搜索兼容时，系统自动回退到原有轻量字段搜索，不会造成旧词条不可检索。
- 后台新增“搜索索引”页面，可查看 SQLite 兼容性、索引覆盖状态、文档数量和更新时间，并由管理员显式执行历史词条回填。
- 新增 `docs/SEARCH_FTS5.md` 与临时 SQLite 自动检查，覆盖中英文匹配、单词条更新和删除同步。
