# Wikist 站点升级日志

## 2026-07-12 - v0.12.5 资讯路由统一

- 移除重复的 `#/news` 资讯壳页面，资讯统一使用词条标准地址 `#/page/news`。
- 顶部导航和首页入口均已改为标准词条路由；旧 `#/news` 与误用的 `#/pages/<slug>` 会自动跳转到 `#/page/<slug>`。
- 框架升级为 `0.12.5`，前端资源版本更新为 `wikist-core-20260712-97`。

## 2026-07-12 - v0.12.4 升级自检稳定性

- 修复 `npm run check:v08` 的验证码测试夹具：不再从随机展示题目反解析答案，而是写入一个隔离、可验证的固定测试验证码。
- 自检仍覆盖注册与验证码校验，但不再因 CAPTCHA 展示格式或随机输入差异误阻断站点升级。
- 框架升级为 `0.12.4`；未改动浏览器资源，前端资源版本保持 `wikist-core-20260712-96`。

## 2026-07-12 - v0.12.3 响应式防护状态

- 平板过渡宽度提前切换为抽屉式站点导航，顶部语言、主题、消息、账户、新建与编辑操作会均匀占满可用宽度，不再被侧栏挤压或与右侧区域重叠。
- 词条质量、收藏、关注、知识链接、组织协作与评分按钮改为自适应满宽网格，窄屏与中间宽度均不再出现左侧堆叠、右侧留白。
- API 返回 `429` 时，Wikist 会锁定当前页面并展示带倒计时的全屏防护状态；到时后才可重新连接，避免导入导出等页面处于半可用状态。
- 单页路由 404 和直接访问未知 API／插件／上传／核心资源时，均会展示统一的 Wikist 科幻状态页，而不再暴露原始 JSON 或纯文本错误。
- 框架升级为 `0.12.3`，前端资源版本更新为 `wikist-core-20260712-96`。

## 2026-07-12 - v0.12.2 健康检查与重启韧性
- `/api/health` 使用独立的高阈值轻量防护策略，不再与普通读取 API 共用计数窗口。
- `run-wikist-server.cmd --restart` 优先通过健康检查确认旧服务，避免内容 API 已被限流时无法原地重启。
- 框架升级为 `0.12.2`，前端资源版本更新为 `wikist-core-20260712-95`。

## 2026-07-12 - v0.12.1 请求防护提示优化

- SPA 内所有 API 请求统一识别 `429` 与 `413`：页面右下角显示带倒计时的防护提示，业务表单得到可读的重试说明。

## 2026-07-12 - v0.12.0 运行可靠性与安装防护
- Passport SQLite 启动统一使用 WAL、`busy_timeout`、foreign keys、自动检查点和 `quick_check` 健康检查；手动下载备份会通过 `VACUUM INTO` 生成一致性快照。
- 备份包升级为带 SHA-256 文件与清单校验的 v2 格式。后台导入会先检查路径、大小、编码与校验和；旧包仍兼容，但会标为 legacy。新增隔离还原演练，不会改动当前站点。
- FTS5 写入或查询失败时，词条保存继续完成，搜索自动回退轻量引擎；后台可单独修复派生索引。
- 新增“运行健康”后台页：展示数据库、搜索索引、请求 / 搜索耗时、缓存命中、插件失败和防护拦截等脱敏聚合指标。
- 新增单机请求防护和安装防火墙：分类限流、请求体上限、`Retry-After`、安装令牌与来源校验。反代环境只有确认可信时才应开启 `trustedProxy`。
- 插件 manifest 支持配置 Schema 和纯声明式迁移；外部服务端模块仍默认不执行，只有已编入核心的受信任 Hook 可在服务端注册。
- 框架升级为 `0.12.0`，前端资源版本更新为 `wikist-core-20260712-93`。
