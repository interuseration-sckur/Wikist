# Wikist

Wikist is a fast, portable, math-first wiki framework for rigorous Chinese knowledge communities.

Release date: 2026-07-10

## What This Repository Contains

This GitHub release is intentionally minimal and install-first. It includes:

- Dependency-light Node.js server code in `src/`.
- Browser UI, installer, styles, and local UI assets in `public/`.
- Trusted plugin manifests and client modules in `plugins/`.
- Documentation in `docs/`.
- Startup and verification scripts in `tools/`.
- Empty runtime directories with `.gitkeep` placeholders.

It does not include local deployment data:

- `data/` SQLite databases, sessions, user accounts, comments, ratings, messages, audit logs, and backups.
- `logs/` and runtime log files.
- `content/pages/`, `content/revisions/`, and `content/deleted/` local wiki pages and history.
- `config/site.config.json`, because it is generated by the installer for each deployment.
- `plugins/vendor/` upstream source caches.

## Requirements

- Node.js 18 or newer.
- Windows, Linux, or macOS shell access.

The runtime is built on Node's standard HTTP server. Passport storage uses Node's `node:sqlite` module on modern Node runtimes. If your Node build does not provide `node:sqlite`, replace `src/core/passport-store.js` with a compatible SQLite adapter before production deployment.

## First Install

Start the server:

```powershell
node server.js
```

On Windows, you can also use:

```powershell
.\run-wikist-server.cmd
```

Then open:

```text
http://127.0.0.1:8899/install.html
```

The installer writes `config/site.config.json` with:

- Site name, tagline, language, and license.
- Portable SQLite path, usually `data/wikist.sqlite`.
- Editing policy.
- Optional SMTP settings for email verification and password reset.

After the installer succeeds, restart the server so the generated configuration is loaded.

## Development Check

```powershell
npm run check
```

The repository also keeps feature check scripts under `tools/`. Some of them create temporary data or assume seeded demo content, so the first-install release only treats `npm run check` as the portable baseline check.

## Storage Model

Wikist separates portable code from deployment state:

- Markdown pages live in `content/pages/`.
- Revision snapshots live in `content/revisions/`.
- Deleted page archives live in `content/deleted/`.
- Passport, comments, ratings, favorites, messages, translations, and audit logs live in SQLite.
- Backups can package content, config, plugin manifests, and optionally user data.

This repository keeps those runtime directories empty by default so each deployment can be installed cleanly.

## Plugin Model

Plugins use local `plugins/*/plugin.json` manifests inspired by MediaWiki extensions.

Supported states:

- `core:*`: trusted behavior implemented by Wikist core.
- `clientModule`: trusted browser module loaded from `/plugins/<plugin>/...`.
- `clone-ready`: upstream repository metadata only; source can be synced into `plugins/vendor/` for review.
- `manifest-only`: catalog, configuration, and documentation only.

Third-party source caches in `plugins/vendor/` are excluded from Git and can be rebuilt with:

```powershell
node tools/sync-vendor-plugins.js
```

## Production Notes

Set a stable Passport secret:

```powershell
$env:WIKIST_PASSPORT_SECRET = "replace-with-a-long-random-secret"
```

To require an edit token:

```powershell
$env:WIKIST_EDIT_TOKEN = "replace-with-an-edit-token"
node server.js
```

For external deployments, place Wikist behind HTTPS and set `WIKIST_HOST=0.0.0.0` only when the host firewall and reverse proxy are configured.
